# Getting started
Once installed enter URL and credentials of your diaspora* pod in the addon preferences.
# Preferences
Pod Url : URL of your diaspora* pod. Enter the full address (https://yourpod.tld). (REQUIRED)
Credentials : the button to enter username/password to sign-in on your pod. (REQUIRED)
From version 0.10.0 the credentials are used to opening a diaspora* session and post to it. They are securely stored in the Firefox Password Manager. You can manage them opening about:preferences#security and clicking the 'Saved Passwords' button.
Bitly OAuth Web Flow : the button to start the authorization process. (Required to shorten URLs)
From version 0.9.0 Easyshare stops using ApiKey authentication to access the Bitly API and switches to OAuth2. Now to shorten URLs you must authorize the addon on your private Bitly account and receive an OAuth access token. The token is stored (hidden) in the preferences and allows Easyshare to access the Bilty API on your behalf.
Display the provider name : a checkbox to add the provider name (via Easyshare) below the post.
# Panel - Url shortening
Use the button near the URL box in the popup dialog to shorten long URLs (prior authorization).
# Panel - Markdown
Use the buttons at the right side of the popup dialog to add a basic formatting to the entered text. You can add markdown syntax for emphasis, strong emphasis or to define an indented quoted section, to the entire text or by selecting part of it.
# Panel - Share with
Enter how to share the post in the input box at right side of the popup dialog. You can choose between:
  • aspect-name : to share with a single aspect entering its name
  • all_aspects : to share with all your aspects
  • public : to share public
N.B. - You can also leave the box blank to share public.

# Tips
- All textarea elements in the popup dialog are resizable to better view their content. To bring them back to their default size just double click on the resizer.
- Markdown buttons for emphasis and strong emphasis don't work if the text contains newline, to avoid a markdown failure. In this case, to enter the syntax you need to select the text, but take care to skip any new line.
- Remember to escape asterisks and underscores in the text that may generate errors when using markdown (like diaspora*).
