DIASPORA* EASYSHARE

Firefox add-on

Extend the right-click context menu with the ability to take content from a web pages. Create the post quickly and easily and send it to diaspora* from every web page.

installable version: 0.16.2
install (require Firefox 57+)

latest version: 0.17.0
GitHub

Due to a bug on AMO, the addon has become unrecoverable and cannot be updated. Getting started

  • Preferences
  • After installing the add-on enter URL and credentials of your pod in the Preferences. Pay attention to enter the absolute URL not only the domain name (e.g. https://joindiaspora.com).
    From here you can also choose a theme for the publisher: Original (default, light) or Dark.
  • Get your aspects
  • The dropdown menu to select aspects initially contains only the default items: Public and All-aspects. Use the button Get Aspects to update it with all your aspects.

Usage

  • The conventional use is via the context menu that allows you to add the content from one or more web pages within the publisher. But you can also simply open the publisher and enter whatever you want.
  • The context menu
  • The context menu item contains two sub-items: one to add the content with a header (the page title and the URL) and the other without (to be used mainly to integrate the saved content).
  • The list of contexts in which the menu item will appear:
    • Image
    • Displayed when you context-click an image in the page. If the image is wrapped with an anchor, the link is preserved.
      Keep in mind that the menu item only appears when clicking on inline images, not on images added via css or javascript.
    • Selection
    • Displayed when you context-click a selection of text in the page. A selection of text can be added within the publisher along with other content.
      Example:
      - select some text and context-click an image.
      N.B. If the selection contains a link, be careful not to click on it because in this case only the link is added to the publisher.
    • Link
    • Displayed when you context-click a link in the page. As mentioned above, this context always takes precedence over selection.
    • Audio/Video
    • Displayed when you context-click html5 audio/video elements in the page. Diaspora has added support for embedding html5 audio/video from version 0.7.4.0.
      Look at the context below (Page) to know on how to add the media URL from Youtube and other media sharing sites.
    • Page
    • Displayed when you context-click any place in the page but none of the other page contexts apply. This add the page URL into the publisher. Use this context to share all those resources - from media sharing sites as Youtube, Vimeo, Soundcloud or Twitter - that are embedded posting their URL.
      What to click in this pages:
      - Youtube: in the video page, context-click anywhere other than the video and other page contexts. Or double click the video.
      - Vimeo: in the video page, context-click the video.
      - Soundcloud: in the track page, context-click the player.
      - Twitter: open a tweet and context-click anywhere other than the video and other page contexts.
  • The publisher
  • The publisher looks like the Diaspora one. You can select an aspect, apply markdown with buttons and preview the content. It automatically popup using the context menu or can be opened with the toolbar button.
    During preview the publisher becomes disabled, you must exit the preview to continue working.
  • You can also:
    • Quick tagging
    • You can insert tags quickly at the end of the post from an input box. Without hash, separated by a comma with or without spaces. But of course you can also insert tags directly into the publisher.
    • Save (content persistent)
    • The popup window that contains the publisher is managed by a WebExtension API. It is created when it is shown and destroyed (with its content) when it is no longer visible. If you want to keep the publisher content persistent, in order to add other content from a different source, use the Save button. The content will be saved in the local storage and automatically entered the next time the popup is opened. The new content taken with the context menu is added below the saved one. The saved content will be automatically deleted when the post is sent to the pod, but you can delete it at any time using the keyboard shortcut Alt+Shift+D. N.B. The Firefox storage space is about 5 MB, sufficient to hold even for very long posts. But if this limit is exceeded an error will be generated.
